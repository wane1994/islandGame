# Домашнее задание: прицел, камера, супервраг и выпадение хилки

Цель задания: самостоятельно улучшить игру про остров и научиться добавлять новые игровые механики через C# скрипты в Unity.

В этом задании нужно сделать четыре вещи:

- прицел для оружия;
- исправить камеру, когда враг находится рядом;
- добавить суперврага, который умирает за 15 попаданий;
- сделать так, чтобы после убийства врага из него выпадала хилка.

Важно: не нужно сразу писать всё идеально. Делай постепенно: сначала маленькая рабочая версия, потом улучшение.

## 1. Подготовка

Перед началом открой проект в Unity и найди скрипты:

```text

Assets/Scripts/IslandPlayerController.cs
Assets/Scripts/IslandPlayerWeapon.cs
Assets/Scripts/IslandEnemy.cs
```

## 2. Задание 1: прицел

### Что нужно сделать

На экране должен быть прицел, чтобы игрок видел, куда полетит пуля.

Прицел должен:

- находиться в центре экрана;
- быть видимым на светлом и тёмном фоне;
- исчезать, когда игра закончилась;
- совпадать с направлением выстрела.

### Где делать

Лучшее место для простого прицела сейчас:

```text
IslandGameManager.cs
```

Там уже рисуется HUD через `OnGUI`.

### Подсказка

Центр экрана можно найти так:

```csharp
float centerX = Screen.width * 0.5f;
float centerY = Screen.height * 0.5f;
```

Прицел можно нарисовать маленькими прямоугольниками через уже существующую функцию `DrawRect`.

Например:

```csharp
DrawRect(new Rect(centerX - 10f, centerY - 1f, 8f, 2f), Color.yellow);
DrawRect(new Rect(centerX + 2f, centerY - 1f, 8f, 2f), Color.yellow);
```

### Что проверить

- Прицел виден во время игры.
- Прицел находится по центру экрана.
- Пуля летит туда, куда показывает прицел.
- После победы или проигрыша прицел не мешает меню.

## 3. Задание 2: камера плохо работает, когда враг рядом

### Проблема

Когда враг подходит близко, камера может смотреть неудобно:

- герой закрывает центр экрана;
- прицел может быть на теле героя;
- враг может оказаться слишком близко к камере;
- игроку трудно целиться.

### Что нужно сделать

Нужно улучшить поведение камеры, чтобы в бою было удобнее.

Минимальная версия:

- камера смотрит не прямо в тело героя, а немного вперёд;
- прицел показывает место перед героем;
- когда враг рядом, камера чуть поднимается или отодвигается.

### Где делать

Основной файл:

```text
IslandPlayerController.cs
```

Там есть:

```csharp
cameraOffset
cameraLookOffset
LateUpdate()
```

### Подсказка 1: смотреть вперёд от игрока

Вместо того чтобы камера смотрела прямо на:

```csharp
transform.position + cameraLookOffset
```

можно повернуть `cameraLookOffset` по направлению игрока:

```csharp
Vector3 lookTarget = transform.position + Quaternion.Euler(0f, yaw, 0f) * cameraLookOffset;
```

Тогда если `cameraLookOffset.z` больше нуля, камера будет смотреть вперёд.

Пример значения:

```csharp
cameraLookOffset = new Vector3(0f, 1.45f, 8f);
```

### Подсказка 2: проверить, близко ли враг

Можно найти врагов:

```csharp
IslandEnemy[] enemies = FindObjectsOfType<IslandEnemy>();
```

Потом проверить расстояние:

```csharp
float distance = Vector3.Distance(transform.position, enemy.transform.position);
```

Если враг близко, например ближе 5 метров, можно изменить камеру:

```csharp
Vector3 closeCombatOffset = new Vector3(0f, 3.8f, -7.5f);
```

### Что проверить

- Когда врагов рядом нет, камера работает как обычно.
- Когда враг рядом, целиться всё ещё удобно.
- Герой не закрывает весь центр экрана.
- Камера не дёргается слишком резко.

Дополнительная сложность: сделай плавный переход камеры через `Vector3.Lerp`.

## 4. Задание 3: супервраг, который умирает за 15 ударов

### Что нужно сделать

Добавить особого врага: суперврага.

Он должен:

- быть заметно больше обычного врага;
- иметь больше здоровья;
- умирать только после 15 попаданий;
- выглядеть другим цветом;
- быть опаснее обычного врага.

### Где делать

Основные файлы:

```text
IslandEnemy.cs
IslandTerrainGenerator.cs
```

### Как посчитать 15 ударов

Если оружие наносит `100` урона, то здоровье суперврага можно сделать:

```csharp
1500
```

Тогда:

```text
1500 HP / 100 damage = 15 попаданий
```

### Подсказка: добавить настройку врага

В `IslandEnemy` можно добавить поле:

```csharp
[SerializeField] private bool isSuperEnemy;
```

И метод настройки:

```csharp
public void ConfigureAsSuperEnemy()
{
    isSuperEnemy = true;
    maxHealth = 1500;
    damage = 24;
    chaseSpeed = 3.4f;
    transform.localScale = Vector3.one * 1.45f;
}
```

Важно: если `currentHealth` задаётся в `Awake`, а `ConfigureAsSuperEnemy()` вызывается позже, нужно также обновить:

```csharp
currentHealth = maxHealth;
```

### Подсказка: создать суперврага в генераторе

В `IslandTerrainGenerator` уже есть код создания врагов.

Можно сделать отдельный метод:

```csharp
private void CreateSuperEnemy(Transform parent)
{
    // выбрать позицию
    // создать врага
    // вызвать enemy.GetComponent<IslandEnemy>().ConfigureAsSuperEnemy();
}
```

И вызвать его после обычных врагов:

```csharp
CreateSuperEnemy(root.transform);
```

### Что проверить

- Супервраг появляется на острове.
- Он больше обычных врагов.
- Он не умирает с одного выстрела.
- Он умирает примерно после 15 попаданий.
- Он не ломает мини-карту и поведение обычных врагов.

## 5. Задание 4: после убийства врага выпадает хилка

### Что нужно сделать

Когда враг умирает, на его месте должна появиться хилка.

Хилка должна:

- появляться рядом с местом смерти врага;
- подбираться игроком;
- восстанавливать здоровье;
- быть видимой на мини-карте, если в проекте уже есть иконка хилки.

### Где делать

Основные файлы:

```text
IslandEnemy.cs
IslandTerrainGenerator.cs
IslandHealItem.cs
```

### Важная идея

Враг не должен сам вручную создавать сложную хилку, если генератор уже умеет создавать хилки.

Лучше сделать в `IslandTerrainGenerator` публичный метод:

```csharp
public void SpawnHealItem(Vector3 position)
{
    // создать сферу
    // добавить trigger collider
    // добавить IslandHealItem
    // добавить IslandMinimapIcon
}
```

Потом в `IslandEnemy.Die()`:

```csharp
IslandTerrainGenerator generator = FindObjectOfType<IslandTerrainGenerator>();
if (generator != null)
{
    generator.SpawnHealItem(transform.position);
}
```

И только после этого:

```csharp
Destroy(gameObject);
```

### Подсказка: поднять хилку над землёй

Если создать хилку прямо в позиции врага, она может оказаться в земле.

Лучше сделать так:

```csharp
heal.transform.position = position + Vector3.up * 1.2f;
```

### Что проверить

- После смерти обычного врага появляется хилка.
- После смерти суперврага тоже появляется хилка.
- Хилка не появляется до смерти врага.
- Игрок может подобрать хилку.
- После подбора хилка исчезает.

## 6. Дополнительные задания

Если основная часть работает, можно усложнить:

- обычный враг иногда роняет хилку, например с шансом 50%;
- супервраг всегда роняет большую хилку;
- хилка мигает или вращается;
- над суперврагом есть полоска здоровья;
- прицел меняет цвет, когда наведен на врага;
- камера плавно отдаляется в бою.

## 7. Главная мысль

В Unity почти любая новая механика делается через связку:

```text
данные -> поведение -> визуальная обратная связь -> проверка в игре
```

Например, для суперврага:

```text
здоровье 1500 -> получает урон -> мигает/умирает -> проверяем 15 попаданий
```

Для хилки:

```text
позиция смерти врага -> создать объект -> игрок подбирает -> здоровье увеличивается
```

Не пытайся сразу сделать идеально. Сначала сделай так, чтобы работало, потом улучшай.
